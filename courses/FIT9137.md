# FIT9137｜Introduction to computer architecture and networks｜计算机体系结构与网络导论

## 写在前面

FIT9137 是很多 Master of IT / Data Science / AI 同学都会遇到的一门计算机底层基础课。

如果你之前不是计算机背景，这门课一开始可能会有点抽象。它不像编程课那样一直写代码，而是更多在帮你理解程序、操作系统、硬件和网络背后到底是怎么运行的。

但它也不是那种“背背概念就能稳”的课。  
我对这门课的感觉是：**内容不一定难写代码，但概念很多，复习压力不小。**

尤其是计算机结构、操作系统、网络协议和系统分层这些内容，前面如果只是听过一遍但没有自己整理，后面做 Project 或 Quiz / Test 的时候会很容易混。

## 课程简介

FIT9137 主要是帮你补计算机底层基础。

这门课会讲计算机硬件、操作系统、网络通信和系统结构。简单说，就是让你知道程序运行时，电脑内部到底发生了什么；网络传输时，数据又是怎么一层一层被处理和发送的。

如果你之后想继续学 Software、Cybersecurity、Cloud、Networks 或者更偏系统方向的课程，这门课都算是比较基础的一块。  
尤其是网络协议和系统结构，后面很多课程都会默认你已经有一些概念。

---

## 主要内容

这门课主要会涉及：

- 计算机硬件基础
- 计算机体系结构
- 操作系统基础
- 内存、CPU 和程序执行逻辑
- 网络通信
- 网络分层
- 常见网络协议
- 系统结构和数据传输逻辑

我觉得最需要认真学的是计算机结构、操作系统和网络分层。  
这些内容不是特别适合死背，最好配合图示理解，不然很容易出现“每个词都见过，但连起来不知道什么意思”的情况。

## Assessment 体感

常见考核结构如下：

- Project 1：20%
- Project 2：45%
- Quiz / Test：35%

FIT9137 的 Assessment 不是单纯考你有没有背下概念。它更偏向“系统理解 + 网络逻辑 + 项目应用”混在一起。

这门课最容易让人误判的地方是：  
刚开始会觉得都是概念，好像看看 slides 就行，但一到 Project 或 Quiz / Test，就会发现很多内容需要你真的理解系统之间的关系。

比如硬件和软件怎么配合、操作系统在中间做什么、网络协议为什么要分层、数据从一台电脑到另一台电脑中间经过了什么，这些都不是靠临时背关键词就能完全解决的。

同学反馈里也会提到 Project 相关的查重 / 相似度检查，所以作业不能随便套模板或者照搬别人的思路。  
即使最后结果看起来差不多，也要确保自己真的理解过程和实现逻辑。

整体来说，它不是最吓人的那种课，但对非计算机背景同学会比较抽象。  
如果前期没有每周整理关键词和图示，后面复习会比较吃力。

---

## 常见卡点

这门课比较容易卡在几个地方：

- 计算机结构太抽象
- 操作系统概念多，容易混
- 网络分层看起来简单，但细节不好记
- 协议之间的作用分不清
- Project 不知道从哪个概念入手
- Quiz / Test 复习范围感觉很散
- 只背关键词，但不会解释整体流程
- 非计算机背景同学容易觉得内容跳得快

我觉得 FIT9137 不是那种靠刷代码就能解决的课。  
它更需要你把系统、硬件、操作系统和网络之间的关系串起来。

---

## 选课搭配和求职方向

FIT9137 看起来是一门底层基础课，但它其实会影响后面很多偏系统和网络方向课程的学习体验。

如果你后面想走 Cybersecurity / Cloud / Networks，计算机结构和网络协议一定会反复出现。  
如果你想走 Software / Backend，理解系统运行和网络通信也会帮助你更好地理解程序背后的环境。

所以这门课建议不要只想着“背过就行”，至少要把计算机结构、操作系统和网络分层的主线理清楚。

公开版先写到这里，后面我会继续补不同方向的选课搭配和避坑组合。

## 学习建议

如果你是非计算机背景，我会建议你不要等到考试前才开始背。

这门课比较适合用“画图 + 关键词整理”的方式学。  
比如学网络的时候，不要只背每一层叫什么，最好自己画一遍数据从应用层到传输层、网络层、链路层是怎么往下走的。

学计算机结构和操作系统的时候，也建议把 CPU、内存、程序、进程这些概念放在一张图里理解。  
很多时候你觉得抽象，是因为这些词单独看都知道，但不知道它们之间怎么连起来。

做 Project 的时候，不要只想着赶结果。  
最好先确认题目到底对应哪一块知识点，再去看要怎么实现或解释。尤其是涉及查重 / 相似度检查的内容，更要确保是自己理解之后写出来的。

如果想稳一点，我觉得核心就是：

- 每周整理关键词
- 多画系统结构图和网络分层图
- 不要只背定义，要会解释流程
- Project 早点开始
- 查重相关内容要特别注意
- 不懂的概念早点问，不要堆到最后
