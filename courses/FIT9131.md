# FIT9131｜Programming foundations in Java｜Java 编程基础

## 写在前面

FIT9131 是很多 Master of IT 同学都会遇到的一门 Java 编程基础课。

如果你之前没有学过 Java，不用一开始就太紧张。这门课会从比较基础的语法开始，慢慢带你理解变量、循环、函数、类和对象这些内容。

但它也不是那种“零基础课 = 很轻松”的课。  
我对这门课的感觉是：**入门是入门，但对零基础同学不算轻松，尤其是 OOP 和 assignment 会比较考验理解。**

Java 和 Python 不太一样，它更强调结构、类型、类和对象。前面如果只是跟着看懂了，但没有自己写代码、debug，后面做 Assignment 或准备考试的时候会比较被动。

## 课程简介

FIT9131 主要是教 Java 编程语言的基础，以及怎么用 Java 写出比较完整的小程序。

这门课会从基础语法开始，慢慢带到面向对象编程 OOP、基础算法逻辑和 debug。简单说，就是让你从零开始理解 Java 代码怎么组织、类和对象怎么用，以及怎么自己写出一个基础的小型应用程序。

如果你之后想走 Software、Backend、Software Engineering 或者项目开发方向，这门课都算是比较重要的基础。  
尤其是 OOP，后面很多软件工程和项目课都会继续用到。

---

## 主要内容

这门课主要会涉及：

- Java 基础语法
- 变量和数据类型
- 条件判断和循环
- 方法 / 函数
- 类和对象
- 面向对象编程 OOP
- 基础算法逻辑
- Debug
- 小型应用程序实现

我觉得最需要认真学的是 OOP、debug 和 assignment 里的代码结构。  
Java 不是只会写几行语法就够了，后面会很看重你能不能把代码组织清楚。

## Assessment 体感

常见考核结构如下：

- Weekly quizzes：10%
- Assignment A + B：45%
- Mid Test：20%
- Exam：25%

FIT9131 的 Assessment 不是单纯看你有没有背语法。它更偏向“Java 基础 + OOP 理解 + 代码实现 + debug 能力”混在一起。

这门课最容易让人误判的地方是：  
刚开始学变量、循环、方法的时候觉得还行，但一到 class、object、constructor、method 之间的关系，就会发现 Java 对结构的要求更强。

Assignment 通常会比 quiz 更考验真实编程能力。  
不是只要写出一段能跑的代码就行，还要看你有没有理解题目要求、类怎么设计、方法怎么拆、bug 怎么排。

整体来说，它不是特别玄的课，但对零基础同学不算轻松。  
如果前期没有跟上 lab、quiz 和 assignment，后面会比较容易被考试和作业一起推着走。

---

## 常见卡点

这门课比较容易卡在几个地方：

- Java 语法和类型不熟
- 循环和条件判断写乱
- 方法不知道怎么拆
- OOP 里的 class / object / method 理解不清楚
- Assignment 不知道从哪里开始写
- Debug 能力不够，报错看不懂
- 代码能跑，但结构不清楚
- 考前才补，发现基础漏洞很多

我觉得 FIT9131 不是最吓人的那种难，但它很吃持续练习。  
尤其是零基础同学，不建议把它当成“考前突击一下就行”的课。

## 选课搭配和求职方向

FIT9131 看起来是一门 Java 入门课，但它其实是后面很多软件方向课程的基础。

如果你后面想走 Software / Backend，Java、OOP、debug 和基础算法逻辑都会反复出现。  
如果你之后要修 Software Engineering、项目课或者更偏开发的 unit，这门课的基础也会影响后面的学习体验。

所以这门课建议不要只想着“过了就行”，至少要把 Java 基础、OOP 和 debug 练明白。

## 学习建议

如果你是零基础，我会建议你提前熟悉一点 Java 基础，不用学得很深，但至少先知道变量、循环、方法和 class 是什么。

这门课比较适合跟着 lab 和 quiz 每周推进。  
不要只看 lecture，因为编程课最怕“看懂了但不会写”。

Assignment 不要拖到最后。  
最好拿到题目之后先读清楚要求，再想清楚需要哪些 class、哪些 method、每一部分负责什么。不要一开始就直接乱写，不然后面 debug 会很痛苦。

准备 Mid Test 和 Exam 的时候，也不要只背概念。  
更重要的是你能不能看懂代码、写出代码、解释代码为什么这样运行。

如果想稳一点，我觉得核心就是：

- Java 基础语法要每周练
- Lab 和 quiz 不要空着
- OOP 要早点理解，不要拖到 assignment 才补
- Assignment 早点开始
- 报错要学会自己读
- 不懂的地方尽早问
