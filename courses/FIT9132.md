# FIT9132｜Introduction to databases｜数据库导论

## 写在前面

FIT9132 是很多 Master of IT / Data Science / AI 同学都会遇到的一门数据库基础课。

如果你之前没学过数据库，不用太紧张。这门课不是一上来就让你做很复杂的系统，而是先带你理解数据库的基本结构：表是什么、表和表之间怎么关联、SQL 怎么查数据、ER 图怎么表达业务逻辑。

但它也不是那种“听听概念就能过”的课。  
我对这门课的感觉是：**入门不难，但细节很多，真正写作业的时候很容易发现自己没想清楚。**

尤其是 SQL、ER 建模、字段设计、表关系，以及从 conceptual model 转到 logical model 这部分，前面如果只是听懂了但没练，后面做 Assessment 会很容易卡。

## 课程简介

FIT9132 主要是教数据库的基本逻辑和实际操作。

这门课会从关系型数据库开始讲，慢慢带到概念模型、ER 建模、SQL 查询、关系代数、范式、数据库设计和数据表管理。简单说，就是让你知道一个数据库应该怎么被设计出来，以及怎么用 SQL 去创建、查询和操作数据。

如果你之后想走 Data、BI、Backend、Software 或者 Business Information Systems，这门课都算是比较基础的一块。  
尤其是 SQL 和数据库设计，后面很多方向都会反复用到。

---

## 主要内容

这门课主要会涉及：

- 关系型数据库基础
- SQL 查询
- 概念模型 conceptual model
- ER 建模
- logical model / relational model
- 概念模型到 logical model 的转换
- 主键、外键和表关系
- 关系代数 relational algebra
- 范式 normalisation
- 数据库设计
- 数据表创建与管理
- 基础数据库操作

我觉得最需要认真学的是 SQL、ER 建模、范式，以及 conceptual model 到 logical model 的转换。  
SQL 是后面做题和写作业的基本工具；ER 建模和 logical model 则更考验你有没有真正理解题目里的业务关系。

## Assessment 体感

常见考核结构如下：

- Quiz / Test 1：15%
- Quiz / Test 2：15%
- Artefact 1 - Database Design：35%
- Artefact 2 - Creating, Populating and Manipulating Databases：35%

FIT9132 的 Assessment 不是单纯背概念。根据 sample 和同学反馈来看，它更偏向“概念理解 + 数据库设计 + SQL 实操”混在一起。

Sample 里会涉及概念模型、关系代数和范式，所以不要只盯着 SQL。  
这门课真正容易卡的地方，往往是你能不能把题目里的业务场景转成正确的数据库结构。

这门课最容易让人误判的地方是：  
刚开始学的时候会觉得“好像都听懂了”，但一到作业或者测试，就会发现很多细节其实没掌握牢。

比如 SQL 写法、字段命名、主键外键、表和表之间的关系、ER 图里的 cardinality、范式判断，以及 conceptual model 怎么转换成 logical model，这些都不是靠背定义就能解决的。  
有些题看起来只是让你画图或者转换模型，但如果前面的实体、关系和约束没判断清楚，后面 logical model 很容易跟着错。

整体来说，它不是知识点特别玄的课，而是要求你真的会用。  
如果前期没有练 SQL 和模型转换，后面会比较被动。

---

## 常见卡点

这门课比较容易卡在几个地方：

- SQL 写得不熟，尤其是 join、group by、subquery 这类内容
- ER 图关系判断不清楚
- conceptual model 到 logical model 的转换容易出错
- 主键、外键和字段设计不规范
- 关系代数看懂了，但自己不会写
- 范式 normalisation 判断不清楚
- 题目里的业务逻辑读不明白
- 作业要求细节看漏
- 以为自己懂了，但真正写的时候不知道怎么下手

我觉得 FIT9132 不是最吓人的那种难，但它很吃细节。  
尤其是零基础同学，不建议把它当成“考前补一补就行”的课。

## 选课搭配和求职方向

FIT9132 看起来是一门数据库入门课，但它其实会影响后面很多课的学习体验。

如果你后面想走 Data / BI，SQL 和数据库一定会反复出现。  
如果你想走 Software / Backend，数据库设计和数据表关系也是基础能力。

所以这门课建议不要只想着“混过去”，至少要把 SQL、ER 建模、范式和 model 转换练明白。

## 学习建议

如果你是零基础，我会建议你从第一周就开始练 SQL，不要只看 lecture。

这门课比较适合用“小题反复练”的方式学。  
比如先把 select、where、join、group by、order by 这些基础写熟，再慢慢看 subquery 和复杂一点的查询。

ER 图和 model 转换也不要只看答案。  
最好自己先根据题目画一遍 conceptual model，再尝试转成 logical model，然后对照 sample 或讲解看自己哪里关系判断错了。很多时候扣分不是因为你完全不会，而是关系、字段、主键外键或者约束条件写得不够准确。

范式和关系代数也建议提前留时间。  
这两部分容易被低估，因为看 lecture 的时候好像能理解，但真到题目里自己写 relational algebra、判断 normalisation，还是很容易卡。

做作业的时候，建议先把题目里的实体、属性和关系圈出来，再去想表怎么设计。  
不要一上来就直接写 SQL，不然很容易后面发现结构不对，又要重改。
