# FIT9132｜Introduction to databases｜数据库导论

## 写在前面

FIT9132 是很多 IT / DS / AI 同学都会遇到的一门数据库基础课。

如果你之前没有学过数据库，不用太慌。这门课不是一上来就让你写很复杂的系统，而是先带你理解数据库到底是什么、表和表之间怎么连接、SQL 怎么查数据。

但也不要因为它是“入门课”就低估它。  
这门课真正容易丢分的地方，往往不是概念听不懂，而是作业细节、SQL 语法和 ER 图设计不够严谨。

## 这门课大概学什么？

简单说，FIT9132 主要是在教你：

- 数据库的基本结构
- SQL 怎么写
- ER 图怎么画
- 表和表之间是什么关系
- 主键、外键为什么重要
- 怎么设计一个相对规范的数据库

如果你以后想走 Data、BI、Backend 或者 Software 方向，这门课都算是比较基础的一块。  
尤其是 SQL，不管你以后做数据分析还是后端开发，都很难完全绕开。

## Assessment 大概是什么类型？

这门课一般会有：

| Assessment 类型 | 大概在做什么 | 容易卡的地方 |
|---|---|---|
| Quiz / Test | 检查基础概念、SQL、数据库关系和设计理解 | 概念会做选择题不代表作业会写 |
| Database Design | 根据 case study 设计数据库结构，通常会涉及 ER 图、表关系、主键/外键 | 关系判断、cardinality、normalisation 容易出错 |
| SQL / Database Implementation | 把设计变成实际数据库，创建表、插入数据、写 SQL 查询或操作数据 | SQL 语法、约束条件、表之间关系容易扣分 |

这里我先不把具体比例和每个作业细节全部放出来，因为每学期可能会有调整，而且细节最好结合当学期 Moodle 看，不过每年的具体内容变化应该不大，我也会等开学后根据具体的形式补充更新。

但先提醒一句：  
这门课的 Assessment 不是单纯背概念，更多是看你会不会真的设计、查询和操作数据库。

## 学姐的真实提醒

我会把 FIT9132 归类成：**逻辑清楚，但细节很多的课**。

它不一定是最难的那种课，但很容易出现这种情况：

“我感觉我听懂了，但一写作业就发现哪里都不太确定。”

比较常见的卡点有：

- SQL 写得不熟
- Join / Group by / Subquery 容易混
- ER 图关系判断不清楚
- 主键、外键设计不规范
- 作业要求看漏，导致一些很细的地方被扣分

所以这门课最不建议的学习方式就是：  
前面觉得简单，后面等 Assessment 快到了才开始补 SQL。

## 选课搭配和求职方向

FIT9132 看起来是一门数据库入门课，但它其实会影响后面很多课的学习体验。

如果你后面想走 Data / BI，SQL 和数据库一定会反复出现；  
如果你想走 Software / Backend，数据库设计和数据表关系也会是基础能力。

所以这门课建议不要只想着“混过去”，至少要把 SQL 和 ER 建模练明白。
